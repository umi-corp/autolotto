# 즉시 구매 설계 (2026-07-17)

사용자 피드백: "자동 예약구매만 되는 것 같은데, 이번주 로또를 못 샀을 때 바로 즉시 구매하는
기능도 있으면 좋겠다." — 예약을 안 해뒀거나 놓친 사용자를 위한 원탭 구매.

## 정책 (확정)

1. **첫 구매 + 추가 구매** — 이번 회차 미구매(`lastPurchasedRound < 현재 회차`)면 저장 슬롯
   그대로 구매. 이미 구매한 회차에서는 버튼이 **"추가 구매"**로 바뀌어 자동 게임 수(1~5)를
   골라 추가 구매할 수 있다(예: 자동구매가 1게임만 샀을 때 남은 주간 한도 4게임 소진).
   추가분은 **자동 게임만** — 저장된 수동 번호를 또 사면 같은 번호 중복 구매가 되기 때문.
   **주간 5게임 한도 판정은 서버에 위임** — 초과 시 동행복권 에러 메시지("주간구매금액 초과")를
   그대로 표시하고, 앱은 로컬 구매수를 추적하지 않는다(앱 밖 웹사이트 구매는 어차피 못 잡음).
2. **번호는 저장된 슬롯 그대로** — 번호 탭에서 설정한 5슬롯(자동/수동)을 그대로 즉시 실행.
   별도 게임 수 선택 UI 없음. 슬롯 미설정이면 번호 설정 화면으로 유도.
3. **자동구매 ON/OFF와 무관** — 로그인 + 슬롯 설정만 있으면 구매 가능. 즉시 구매는 예약의
   보완이 아니라 독립 기능.

## UX

위치: 홈 화면, 자동구매 상태 카드 아래. **"지금 바로 구매"가 프라이머리 CTA**(네이비
CtaButton), 기존 "번호 설정"은 아웃라인 버튼으로 그 아래 유지.

버튼 상태 (위에서부터 우선):

| 상태 | 표시 |
|---|---|
| 비로그인 | 비활성 + "로그인 후 이용 가능" |
| 판매시간 외 (매일 00~06시, 토 20시~일 06시, KST) | 비활성 + "지금은 판매시간이 아닙니다" |
| 이번 회차 구매 완료 | 활성 — "추가 구매" (자동 게임 수 선택 플로우) |
| 슬롯 미설정 | 활성 — 탭 시 번호 설정 화면 안내 |
| 정상 | 활성 — "지금 바로 구매" |

첫 구매 플로우: 탭 → 확인 다이얼로그("제 N회 · M게임 · ₩M,000을 지금 구매할까요?" — 실결제라
원탭 즉시 결제 금지) → 진행 스피너 → 성공 시 결과 다이얼로그(구매 번호를 LottoBall로 표시) +
잔액 자동 갱신. 이후 같은 회차 예약구매는 기존 가드가 자동 스킵.

추가 구매 플로우: 탭 → 자동 게임 수 선택 다이얼로그(1~5, 금액 표시, 그 자체가 확인 다이얼로그)
→ 진행 스피너 → 결과/에러는 첫 구매와 동일. 한도 초과면 서버 에러 메시지 그대로.

실패: 서버 메시지(주간구매금액 초과 등) 그대로 에러 다이얼로그. **자동 재시도 없음** —
AutoPurchaseWorker와 같은 이유(서버가 이미 처리했을 수 있어 재전송은 중복 결제 위험).
수동 재시도는 가능하며 성공 건은 가드가 차단.

세션: 탭 시점에 앱 세션이 만료됐을 수 있으므로 구매 직전 저장된 자격증명으로 **재로그인 후
구매**(워커와 동일 패턴). 세션 만료 실패 원천 제거, 비용은 1~2초.

## 구현 (변경 4곳, 새 파일 없음)

1. **AppContainer.kt** — `suspend fun instantPurchase(autoGames: Int, manualNumbers): PurchaseResult`:
   자격증명 읽기 → `auth.login()` → `PurchaseService.purchase()` → 성공 시
   `setLastPurchasedRound()` + `refreshBalance()`(잔액부족 알림 포함, 워커와 동일).
   무엇을 살지(슬롯 vs 추가 자동 N게임)는 호출자(VM)가 결정. `lastPurchasedRound`는
   StateFlow로 노출하고 스플래시 하이드레이션·구매 성공·홈 새로고침 때 갱신(워커가
   백그라운드에서 갱신한 경우 대비).
2. **ViewModels.kt (HomeViewModel)** — `InstantState`(Idle/NeedsSetup/Confirming/
   PickingExtra/InProgress/Success/Error) StateFlow + 탭·확인·닫기 액션. 첫 구매 탭은
   슬롯을 읽어 Confirming으로, 구매완료 회차 탭은 PickingExtra(게임 수 선택)로 분기.
   판매시간·구매완료 게이트는 **탭 시점 재검증**(표시용 계산은 구성 시점이라 자정 넘김
   등에서 stale 가능). 판매시간 판정은 회차 계산과 동일하게 KST 고정,
   `SettingsViewModel.isValidPurchaseTime` 재사용.
3. **HomeScreen.kt** — CTA 영역 교체(즉시구매/추가구매 프라이머리 + 번호설정 아웃라인),
   확인·게임수선택·결과·에러 다이얼로그, 상태별 비활성 라벨. 미설정 슬롯 안내는 기존
   `onNavigateToNumbers` 재사용.
4. **strings.xml ko/en/ja** — 버튼·다이얼로그·상태 라벨 ~8개 × 3언어.

건드리지 않는 것: AutoPurchaseWorker, PurchaseService, SecureStore(키 기존), 알람 스케줄러.
즉시 구매가 남긴 `lastPurchasedRound`를 워커 가드가 읽으므로 예약구매 중복 방지는 공짜.

## 검증

컴파일 + assembleDebug → 에뮬레이터에서 4개 비활성 상태(비로그인/구매완료/시간외/미설정) 표시
확인. 실결제 E2E(1게임 ₩1,000)는 실계정 필요 — 배포 전 사용자 계정으로 1회 여부는 별도 확인.
