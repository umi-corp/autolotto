# 번호 탭 인터랙션 시나리오 (2026-07-11)

> **상태: P1~P6 전부 적용 완료 (2026-07-11).** 아래 🔴/🟡 표기는 분석 시점 기준 기록.

## 0. 상태 모델

```
슬롯(A~E) 5개, 각각 3상태:  미설정(null) │ 자동([]) │ 수동([6개 번호])
```

| 층 | 무엇 | 반영 시점 |
|---|---|---|
| **편집기** (토글·그리드·선택영역) | 현재 슬롯의 "편집 중" 뷰 (`isAuto`, `selected`) | 즉시 (화면에만) |
| **슬롯** (`games[5]`) | 확정된 게임 구성 (메모리) | **확정 / 전부 자동 / 초기화 / X** 시 |
| **저장소** (SecureStore) | 자동구매가 실제로 읽는 원본 | **저장** 버튼 시에만 |

핵심 원칙 (이번 수정으로 확립): **슬롯 진입 시 편집기는 그 슬롯의 저장 상태로 동기화된다** (`syncEditorToSlot`) — 자동 슬롯이면 토글=자동, 수동 슬롯이면 볼 6개 활성 표시, 미설정이면 빈 회색 그리드.

수명: 하단 탭(홈↔번호 등) 전환은 **상태 유지**(페이저 전체 상주). 앱 재시작 시에만 저장소에서 복원 → 미저장 편집은 유실.
